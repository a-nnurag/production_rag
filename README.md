# 🎯 Overview

This system enables users to query knowledge sources and receive context-aware responses generated by Large Language Models (LLMs).

* Prompt Injection Attacks
* PII Leakage
* High Latency
* Expensive LLM Calls
* Lack of Monitoring

* API Abuse

## 🔐 Security First Design

### Prompt Injection Protection

Detects and blocks malicious instructions such as:

* Ignore previous instructions
* Reveal system prompt
* Bypass safety rules
* Act as administrator
* Execute hidden commands

Features:

* Injection pattern detection
* Input sanitization
* Security validation layer
* Request rejection mechanism
* Security event logging

---

### 🛡️ PII Detection & Masking

Automatically identifies and masks sensitive information before processing.

Supported detections include:

* Email Addresses
* Phone Numbers
* API Keys
* Access Tokens
* Secret Credentials
* Government IDs
* Credit Card Patterns

Example:

Input:

[user@gmail.com](mailto:user@gmail.com)

Output:

[EMAIL_REDACTED]

# 🧪 Testing Strategy

This project follows **Test-Driven Development (TDD)** principles.

Development workflow:

Red → Green → Refactor

1. Write failing test
2. Implement functionality
3. Pass test
4. Refactor safely

---

## AAA Testing Pattern

All tests follow the Arrange-Act-Assert methodology.

Example:

```python
def test_email_detection():

    # Arrange
    detector = PIIDetector()
    text = "user@gmail.com"

    # Act
    result = detector.detect(text)

    # Assert
    assert result.has_pii is True
```
